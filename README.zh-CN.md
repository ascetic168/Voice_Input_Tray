# Voice Input Tray

**[正體中文](README.md)** | **[简体中文](README.zh-CN.md)** | **[English](README.en.md)**

> 语音识别输入工具 — 纯 CPU 即可流畅运行，无需 GPU。可全本地、无云端运行（适合企业或保密机关内部使用），低延迟串流识别 + SenseVoice 文字修正

## 项目状态

✅ **已发布 v0.4.0** — 新增倾听模式：连按两下右 Ctrl 键开启，电脑将连续不断倾听并辨识声音，直到关闭为止。

## 为什么使用这个程序

与市面上其他语音输入工具相比，Voice Input Tray 解决了三个核心痛点：

### 1. 全本地运行，零隐私疑虑，无需 GPU
所有语音识别都在本机完成，音频不会上传到任何服务器。采用串流 Zipformer 即时识别搭配 SenseVoice 离线修正的双引擎架构，**纯 CPU 即可流畅运行，无需独立显卡或 CUDA 环境**，无需网络、无需 API 密钥，下载模型后即可离线使用。

### 2. 串流 + 离线双引擎协作，即时反馈零延迟
独创的串流-离线协作架构：
- **串流引擎**（Zipformer）：按下热键的瞬间即开始即时识别，边说边显示结果
- **离线引擎**（SenseVoice）：放开热键后，对完整音频进行高精度识别，自动替换串流结果
- 串流引擎负责即时反馈，离线引擎负责最终品质，两者无缝协作

### 3. 单键 CTRL，智能判断使用意图
只需按住 Ctrl 键即开始录音，程序会自动判断您的使用意图：
- 按下 Ctrl 后 2 秒内按其他键 → 自动取消，不影响 Ctrl+C、Ctrl+V 等快捷键
- 过短的录音（< 0.8 秒）自动丢弃，避免误触产生垃圾文字
- 贴上后自动恢复原有剪贴簿内容，不影响使用者原有的复制贴上工作流

## 功能特色

- **全本地运行** — 串流 Zipformer + SenseVoice 双引擎，全程离线，零隐私疑虑
- **纯 CPU 即可流畅运行** — 无需 GPU、无需 CUDA，一般笔电或桌机都能即时识别
- **串流即时反馈** — 边说边显示识别结果，Overlay 浮动窗口即时呈现
- **SenseVoice 品质修正** — 放开热键后自动用高精度离线引擎替换串流结果
- **SenseVoice 内置标点** — 启用 ITN 功能直接输出。，？等标点符号，无需额外模型，识别速度更快
- **语音活动检测** — Silero VAD 自动过滤静音停顿，消除识别垃圾字符（可调整灵敏度）
- **离线识别** — 无需网络及 API 密钥，首次启动自动下载模型（含 VAD 模型）
- **中国大陆镜像** — 自动检测系统语言，简体中文环境使用 hf-mirror.com 加速下载
- **多引擎 ASR** — 离线引擎（默认）、腾讯云 ASR、Google Gemini，可自定义优先顺序
- **自动 Fallback** — 设定优先顺序，前一引擎失败自动尝试下一个
- **跨平台支持** — 支持 Windows、macOS、Linux
- **快捷键触发** — 按住 Ctrl 键即可开始录音，放开自动识别并贴上
- **倾听模式** — 连按两下右 Ctrl 键开启，连续不断倾听并辨识声音，直到关闭为止
- **系统托盘运行** — 背景运行，不干扰日常工作流程
- **繁简转换** — 自动根据系统地区转换中文繁简体，串流与最终结果皆适用
- **多语言 UI** — 支持繁體中文、简体中文、English 介面
- **开机自启** — 支持系统开机自动启动功能
- **剪贴簿保护** — 贴上后自动恢复原有剪贴簿内容

## 支持的 ASR 引擎

### 本地双引擎（默认）
- **串流 Zipformer**（即时）— 中英双语串流识别，边说边显示
- **SenseVoice-Small (INT8)**（离线修正）— 高精度离线识别，支持中英日韩语，内置 ITN 直接输出标点
- 特色：全程离线、零隐私疑虑、首次启动自动下载模型
- 模型储存位置：`C:\ProgramData\VoiceInput\models\`（Windows）

### 腾讯云 ASR
- **SentenceRecognition API** - 同步识别
- 支持引擎：16k_zh、16k_zh_video、16k_en、16k_yue、16k_ca
- 需要：Secret ID + Secret Key

### Google Gemini
- **Multimodal generateContent API** - AI 加持语音识别
- 支持模型：gemini-2.5-flash、gemini-2.5-flash-lite
- 自动标点符号、移除语气词、可指定输出语言
- 需要：API Key

## 快速开始

### 安装

请至 [Releases 页面](https://github.com/ascetic168/Voice_Input_Tray/releases) 下载安装档。

### 首次启动

1. 启动后自动开启设定窗口
2. 点击「下载模型」按钮，自动下载串流、离线识别模型
3. 下载完成后即可使用，无需设定 API 密钥

### 使用方式

1. 按住 Ctrl 键开始说话（Overlay 浮动窗口即时显示串流识别结果）
2. 放开 Ctrl 键，SenseVoice 自动进行高精度修正并加上标点符号
3. 识别结果自动贴上到当前应用程序
4. 原有剪贴簿内容自动恢复

## 技术架构

```
使用者按下 Ctrl
       ↓
  ContinuousRecorder (cpal 持续录音流，48kHz 立体声)
       ↓
  音频 chunks → 降混 mono → 降采样 16kHz
       ↓
  ┌──────────────────────────────────────────┐
  │       Silero VAD 语音活动检测            │
  │  Pre-buffer 机制保留语音开头上下文        │
  │  静音片段自动过滤，不送入识别引擎         │
  └──────────────────────────────────────────┘
       ↓ (语音 chunks only)
  ┌──────────────────────────────────────────┐
  │          串流识别线程 (Zipformer)         │
  │  即时产生 partial text → Overlay 显示     │
  │  Endpoint 检测 → 片段音频送 SenseVoice   │
  └──────────────────────────────────────────┘
       ↓ (放开 Ctrl)
  StreamingFinished → 剩余音频送 SenseVoice
       ↓
  ┌──────────────────────────────────────────┐
  │        SenseVoice 修正线程 (离线)         │
  │  高精度识别 → 取代串流结果               │
  └──────────────────────────────────────────┘
       ↓
  SenseVoice 内置 ITN 输出标点（自动加上。，？等标点）
       ↓
  convert.rs (繁简转换，依系统地区)
       ↓
  clipboard.rs (保存原剪贴簿 → 设定文字 → Ctrl+V → 恢复剪贴簿)
```
